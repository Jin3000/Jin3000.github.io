一.maven

1.理解分模块开发的意义是什么？

将原始模块按照功能拆分成若干个子模块，方便模块间的互相调用，接口共享。

  • 代码组织和管理：
    • 将一个大型项目拆分成多个模块，每个模块负责不同的功能或业务逻辑，使得代码结构更加清晰。例如，在一个电商系统中，可以将用户模块、商品模块、订单模块、支付模块等拆分为不同的模块，这样不同的开发团队可以专注于自己负责的模块，提高开发效率和代码的可维护性。
    • 方便代码的查找和定位，当需要修改或添加功能时，能迅速找到对应的模块，而不是在一个庞大的项目中寻找。
  • 模块复用：
    • 可以将一些通用的功能封装在一个模块中，供其他项目或本项目的不同部分复用。比如将常用的工具类封装在一个工具模块中，多个业务模块可以引用这个工具模块，避免重复开发，提高代码的复用性和一致性。
    • 促进了代码的解耦，不同模块之间通过明确的接口进行交互，降低了模块间的耦合度，当一个模块发生变化时，对其他模块的影响可以控制在最小范围内。

2.如何分模块开发呢？分模块开发的步骤？

①创建Maven模块

创建模块目录结构：

  • 在项目的目录下，创建不同的子目录作为模块的根目录。在 Maven 中，可以使用 mvn archetype:generate 命令创建一个新的 Maven 模块，或者手动创建目录结构，包括 src/main/java（存放 Java 源文件）、src/main/resources（存放资源文件）、src/test/java（存放测试文件）、pom.xml（模块的 Maven 配置文件）等。
  • 在 pom.xml 文件中配置模块的基本信息，如模块名称、版本、依赖关系等。

②书写模块代码（注：分模块开发需要先针对模块功能进行设计，再进行编码。不会先将工程开发完毕，然后进行拆分）

  • 配置模块依赖：
    • 在每个模块的 pom.xml 中使用 <dependencies> 元素添加该模块所需的依赖。例如，如果订单模块需要使用用户模块的服务，在订单模块的 pom.xml 中添加用户模块的依赖。
    • 对于依赖的范围，根据需要选择 compile（编译、运行、测试时都需要）、test（仅测试时需要）、runtime（运行时需要）等。
  • 开发模块功能：
    • 在 src/main/java 目录下编写模块的具体代码，实现模块的功能。
    • 遵循一定的代码规范和设计模式，保证代码的质量和可维护性。
  • 模块集成：
    • 在父 pom.xml 中使用 <modules> 元素将所有的子模块包含进来，确保 Maven 能够将所有模块作为一个整体项目进行构建。
    • 当构建父项目时，Maven 会根据模块之间的依赖关系自动编译、打包子模块，并将它们组装在一起。

③通过maven指令安装模块到本地仓库（install指令）（注：团队内部开发需要发布模块功能到团队内部可共享的仓库中（私服））

YML

  • YML（YAML）是一种数据序列化语言，其特点是使用缩进和换行来表示数据结构，可读性强。
    • 大小写敏感
    • 属性层级关系使用多行描述，每行结尾使用冒号结束
    • 使用缩进表示层级关系，同层级左侧对齐，只允许使用空格（不允许用tab键）
    • 属性值前面添加空格（属性名与属性值之间使用冒号+空格作为分隔）
    • # 表示注释
    • 核心规则：数据前面要加空格与冒号隔开
