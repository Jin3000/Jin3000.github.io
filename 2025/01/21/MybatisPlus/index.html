5.逻辑删除？为什么不直接在数据库中删除？

  • 逻辑删除：为数据设置是否可用状态字段，删除时设置状态字段为不可用状态，数据保留再数据库中。

    • 逻辑删除是对数据进行标记，真正的数据不可能从业务数据库中删除。

  • 删除操作业务问题：业务数据从数据库中丢弃

6.id生成的几种策略？

  • 雪花算法：默认的主键生成策略，生成全局唯一的 64 位长整型(long) ID，保证分布式系统下 ID 的唯一性。

  • 自增策略（AUTO）：使用数据库的自增功能生成主键，需数据库支持自增，在实体类上添加 @TableId(type = IdType.AUTO) 注解。
  • 输入 ID：需要开发者手动输入 ID，在实体类上添加 @TableId(type = IdType.INPUT) 注解。
  • ASSIGN_ID：适用于分布式系统，当实体类主键类型为 Long 或 String 时，会根据雪花算法生成主键，添加 @TableId(type = IdType.ASSIGN_ID) 注解。
  • ASSIGN_UUID：使用 UUID 作为主键，适用于主键为 String 类型的情况，添加 @TableId(type = IdType.ASSIGN_UUID) 注解。
