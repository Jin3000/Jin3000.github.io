Mysql数据库

1.数据库数据类型，并指出这些类型对应Java里面的哪些数据类型。

  • 整数类型：
    • TINYINT：在 Java 中可以使用 byte 类型存储。
    • SMALLINT：对应 short 类型。
    • INT：可以使用 int 类型存储。
    • BIGINT：对应 long 类型。
  • 浮点数类型：
    • FLOAT：在 Java 中使用 float 类型存储，不过在使用时需要注意精度问题。
    • DOUBLE：使用 double 类型存储。
    • DECIMAL：通常可以使用 java.math.BigDecimal 类存储，因为它可以精确表示十进制数，适合存储货币等需要精确表示的数值。
  • 字符串类型：
    • CHAR：在 Java 中可以使用 String 存储，其长度固定，存储时会使用空格填充到指定长度。
    • VARCHAR：使用 String 存储，长度可变，节省空间。
    • TEXT：一般使用 String 存储，适用于存储较长的文本内容。
  • 日期和时间类型：
    • DATE：可以使用 java.sql.Date 类存储日期部分。
    • TIME：使用 java.sql.Time 存储时间部分。
    • DATETIME：使用 java.sql.Timestamp 存储日期和时间。
    • TIMESTAMP：使用 java.sql.Timestamp 存储，存储从 1970-01-01 00:00:00 开始的秒数，受时区影响。
  • 布尔类型：
    • BIT：在 MySQL 中可以存储布尔值，在 Java 中可以使用 boolean 或 Boolean 类存储。

2.InnoDB存储引擎的特点？

  • 事务支持：InnoDB 是 MySQL 的默认存储引擎，它支持事务，遵循 ACID 特性（原子性、一致性、隔离性、持久性），确保数据的完整性和一致性。
  • 行级锁：使用行级锁，相对于表级锁，可以提高并发性能，多个事务可以同时操作不同行，减少锁冲突。
  • 外键约束：支持外键，方便维护表之间的关系，保证数据的参照完整性。
  • 崩溃恢复：在系统崩溃或异常关闭后，可以通过事务日志进行数据恢复，确保数据不会丢失或损坏。
  • 自动增长列：支持自动增长列（AUTO_INCREMENT），方便创建唯一的标识符。
  • 多版本并发控制（MVCC）：使用 MVCC 来实现事务的隔离级别，允许并发事务读取数据的不同版本，提高并发性能。

5.熟悉数据库运算符号

  • 算术运算符：
    • +：加法运算，例如 SELECT 5 + 3; 结果为 8。
    • -：减法运算，例如 SELECT 10 - 2; 结果为 8。
    • *：乘法运算，例如 SELECT 4 * 5; 结果为 20。
    • /：除法运算，例如 SELECT 10 / 2; 结果为 5。
    • %：取模运算，例如 SELECT 10 % 3; 结果为 1。
  • 比较运算符：
    • =：等于，例如 SELECT * FROM students WHERE age = 20;。
    • >：大于，例如 SELECT * FROM students WHERE age > 20;。
    • <：小于，例如 SELECT * FROM students WHERE age < 20;。
    • >=：大于或等于，例如 SELECT * FROM students WHERE age >= 20;。
    • <=：小于或等于，例如 SELECT * FROM students WHERE age <= 20;。
    • <> 或 !=：不等于，例如 SELECT * FROM students WHERE age <> 20;。
  • 逻辑运算符：
    • AND：逻辑与，例如 SELECT * FROM students WHERE age > 20 AND grade = 'A';。
    • OR：逻辑或，例如 SELECT * FROM students WHERE age > 20 OR grade = 'A';。
    • NOT：逻辑非，例如 SELECT * FROM students WHERE NOT age > 20;。

10.索引使用的什么数据结构，为什么采用这种数据结构不采用B树？什么是索引下推？什么是回表？

  • 索引的数据结构：
    • 通常使用 B+ 树作为索引的数据结构。B+ 树是一种平衡多路查找树，具有以下优点：
      • 所有叶子节点形成一个有序链表，便于范围查询，能有效支持 ORDER BY、BETWEEN 等操作。
      • 非叶子节点只存储索引信息，不存储数据，能存储更多的索引，减少树的高度，提高查询效率。
    • 不使用 B 树的原因：B 树的叶子节点和非叶子节点都存储数据，存储索引信息的空间相对较少，对于范围查询等操作不如 B+ 树高效。
  • 索引下推（Index Condition Pushdown，ICP）：
    • 在存储引擎层，将部分本应在服务器层进行的条件判断下推到存储引擎层，减少存储引擎返回给服务器的数据量，提高性能。例如，在使用组合索引时，如果查询条件包含索引的部分列，存储引擎可以利用索引下推先过滤部分数据，减少不必要的数据传输。
  • 回表：
    • 当使用二级索引（非主键索引）进行查询时，如果查询的列不在该索引中，存储引擎会先通过二级索引找到对应的主键，再通过主键索引找到所需的数据行，这个过程称为回表。
